# Карта стейкхолдеров для уточнения требований

| Роль | Зона ответственности | Какие вопросы уточнить | Приоритет |
| :--- | :--- | :--- | :--- |
| Спонсор проекта (директор по цифровым продажам) | Бизнес-цели, приоритеты, ограничения, ожидаемый эффект | Бюджет, сроки, метрики успеха, приоритеты между стабильностью и скоростью, допустимый уровень риска | Высокий |
| Руководитель команды заказов и оформления | Реальный процесс оформления, исключения, ручные операции, операционные проблемы | Детальные сценарии оформления, частота исключений, зависимости от других команд, «болевые точки» в коде и интеграциях | Высокий |
| Технический лидер монолитного ядра | Фактическое устройство системы, technical ограничения, узкие места, долг | Архитектура кода, схемы БД, точки расширения, legacy-решения, производительность, наблюдаемость | Высокий |
| Руководитель клиентской поддержки | Клиентские проблемы, непрозрачность статусов, ручные операции | (Уже есть стенограмма; можно дополнить вопросами по метрикам SLA, объёму обращений) | Средний |
| Руководитель интеграций / партнёров | Внешние системы (маркетплейсы, платежи, доставка) | Протоколы, частота обновлений, проблемы синхронизации, SLA внешних систем | Средний |
| Инфраструктурный инженер / администратор БД | On-premise инфраструктура, Oracle, производительность БД | Ресурсы, лимиты, планы расширения, проблемы с архивированием, производительность запросов | Средний |
| Бизнес-пользователи (маркетинг, логистика, финансы) | Требования к данным, отчётности, управленческим процессам | Расхождения в данных, потребность в аналитике, частота отчётов | Низкий (на поздних этапах) |
