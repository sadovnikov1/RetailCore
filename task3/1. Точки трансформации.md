# 1. Точки трансформации
На основе анализа текущего состояния выделены 5 точек трансформации, которые могут быть реализованы поэтапно с приемлемым уровнем риска. Каждая точка описывает: проблему, решение, зависимости, риски и предполагаемый порядок реализации.

# Точка 1. Выделение событийного слоя заказа (Order Event Stream)

| Аспект | Описание |
|--------|----------|
| **Проблема** | Статусы и изменения заказа разбросаны по нескольким системам (монолит, .NET, Oracle, batch). Нет единого потока событий для аналитики, поддержки и интеграций. Рассинхрон статусов приводит к непрозрачности для клиента и ручным эскалациям. |
| **Решение** | Внедрить Kafka как единую шину событий заказа. Монолит при каждом значимом изменении (создание, оплата, смена статуса, отмена, изменение доставки) публикует событие в топик `order-events`. <br><br> **Потребители:** <br> • Аналитика (ClickHouse/Data Warehouse) <br> • Поддержка (обновление карточки заказа) <br> • .NET-интеграции (замена staging-таблиц) <br> • Логистика (обновление статусов) |
| **Почему этот участок подходит для изменений** | Это **read-side** изменения — публикация события не изменяет критическую логику checkout. Можно внедрять постепенно: начать с публикации событий в дополнение к существующим механизмам (dual-write), затем переключать потребителей. |
| **Зависимости** | • Требуется инфраструктура Kafka (managed service в облаке или self-hosted) <br> • Нужно доработать монолит для публикации событий (добавить продюсеры в ключевых точках) <br> • Потребители (.NET, Python) должны быть адаптированы |
| **Риски** | • **Риск потери событий при сбое Kafka** — использовать `acks=all` и репликацию <br> • **Риск дублирования** — внедрить idempotent producers и уникальные `event_id` <br> • **Риск увеличения латенции checkout** — публикация должна быть асинхронной (не блокировать транзакцию) |
| **Ожидаемый эффект** | • Единый источник правды по изменениям заказа <br> • Улучшение наблюдаемости <br> • Снижение ручных операций <br> • Основа для выноса аналитики и интеграций |
| **Порядок реализации** | **1-й этап (0–3 месяца)** |
